# ctct project explanation

CTCT project explanation

When it comes to my project, I was working for a US client, CTCT, which is an email marketing company. I was a shared resource between the Cloud team and the app team, mainly working on the Cloud team, where I received tickets from various teams. On a daily basis, I would get different tickets for different tasks. For example, some teams needed updates to Jenkins file management, modifications to their pipeline, or help when a pipeline was broken. I would take care of the complete pipeline modification and debugging of issues.

Another example: some teams may need modifications to the Terraform infrastructure, or they may need additional infrastructure. They would raise a request for the Cloud team to modify their infrastructure, so I would take up the ticket and work on the Terraform configuration files that needed to be modified. I would deploy their infrastructure.

When it comes to the development activities with respect to the team, I was taking complete responsibility, where I was responsible for 3 applications: 2 running on ECS and 1 on-prem application running on OpenStack.

BSS (Billing Shared Services team) is mainly responsible for billing-related applications. Within that, we have multiple applications like PaaS (payment as a service), NaaS (notification as a service), and BaaS (billing as a service).

Under this, the infrastructure includes BaaS on an on-prem service (OpenStack), and PaaS and NaaS as AWS services.

For the AWS services, we used CodePipeline, CodeBuild, CloudFormation, ECS service, and Route 53 to host our application. These are the main services that were used in that application. Both of these applications are running in ECS as containers.

I took complete ownership of these applications, from the beginning of DevOps development till the end of production support.

As I said, I was part of the development team as well as the support team, where I would take multiple tickets from various teams to support multiple teams as well.

We are mainly using Jira to handle our tickets and work managements.

This is just a brief introduction about my roles and responsibilities with respect to my last project.
